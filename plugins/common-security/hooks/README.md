# common-security hooks

`common-security/hooks`는 보안 또는 운영 안전에 영향을 줄 수 있는 작업 전후에 적용할 공통 제한 규칙을 정의합니다.

현재는 실제 실행 Hook 스크립트가 아니라, 향후 Hook으로 전환할 수 있는 기준 문서로 관리합니다.

## 구성

| Hook | 파일 | 역할 |
|---|---|---|
| `dangerous-operation-guard` | `dangerous-operation-guard.md` | 위험 작업 실행 전 확인 규칙 |

## dangerous-operation-guard

운영 환경, 데이터 삭제, DB 변경, 부하 테스트, 배포, 권한 변경과 같은 위험 작업 전에 확인해야 하는 기준입니다.

## Hook 적용 후보

향후 실제 Hook으로 전환할 수 있는 대상은 다음과 같습니다.

- 운영 DB 변경 명령 감지
- 운영 Redis 변경 명령 감지
- 삭제성 명령 감지
- 부하 테스트 명령 감지
- 배포 명령 감지
- Secret 포함 파일 커밋 감지

## 작성 원칙

- 초기에는 자동 차단보다 명시적 경고와 확인 기준 문서화에 집중합니다.
- 실제 Hook으로 전환할 때는 오탐 가능성과 개발자 경험을 함께 고려합니다.
- 운영 환경 영향 가능성이 있는 작업은 사용자 승인 없이 실행하지 않습니다.
