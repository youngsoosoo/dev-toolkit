# common-documentation hooks

`common-documentation/hooks`는 문서화 작업 전후에 적용할 수 있는 공통 검증 기준을 관리하는 위치입니다.

현재는 실제 실행 Hook을 두지 않습니다.

## 현재 상태

| Hook | 상태 | 설명 |
|---|---|---|
| 문서 품질 Hook | 미사용 | 문서 품질은 `documentation-reviewer` Agent로 검토합니다. |
| 민감정보 Hook | 미사용 | 민감정보 검토는 `common-security` 기준을 우선 적용합니다. |

## 향후 Hook 적용 후보

향후 필요하면 아래 기준을 Hook으로 전환할 수 있습니다.

- 문서에 민감정보가 포함되어 있는지 확인
- 문서에 `미검증` 항목 없이 완료 처리하려는 경우 경고
- 릴리즈 노트에 롤백 기준이 없는 경우 경고
- 장애 보고서에 원인/영향/조치/후속 작업이 누락된 경우 경고

## 운영 원칙

- 초기에는 자동 차단보다 문서 리뷰 Agent를 통한 품질 검토를 우선합니다.
- 문서화 Hook은 오탐 가능성이 크기 때문에 신중하게 도입합니다.
- 보안/개인정보 관련 검증은 `common-security` 기준을 우선합니다.
