# Weekly Report

This skill writes a concise weekly work report for team sharing or management reporting.

## When to use

Use this skill when:

- The user wants to summarize weekly work.
- Multiple tasks need to be grouped by theme or project.
- Progress, blockers, risks, and next plans need to be communicated.

## Output format

Use the following structure:

```md
## 이번 주 진행 업무

## 완료한 작업

## 진행 중인 작업

## 이슈 및 리스크

## 다음 주 계획

## 공유/확인 필요 사항
```

## Rules

- Keep the report concise and business-readable.
- Group related tasks instead of listing every small action.
- Mark uncertain or pending items as 확인 필요.
- Do not exaggerate impact.
- Include blockers or risks if they affect schedule or delivery.
