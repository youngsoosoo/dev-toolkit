# common-documentation

`common-documentation`은 모든 직군이 공통으로 사용할 수 있는 문서화 기준과 템플릿을 정의하는 Claude Code 플러그인입니다.

이 플러그인은 특정 기술 스택의 구현 방식이 아니라, 작업 결과를 일관된 형식으로 기록하고 공유하기 위한 문서화 절차를 다룹니다.

## 목적

`common-documentation`의 목적은 다음과 같습니다.

- 작업 완료 내용을 일관된 형식으로 정리한다.
- 주간 업무 보고, 릴리즈 노트, 장애 보고서, 의사결정 기록을 표준화한다.
- 문서에 확인된 사실, 미검증 항목, 후속 작업을 명확히 남긴다.
- 개발자, 기획자, 운영자, 리뷰어가 같은 맥락을 이해할 수 있도록 한다.
- 과장되거나 검증되지 않은 완료 표현을 줄인다.

## 구성

| 디렉터리 | 역할 |
|---|---|
| `skills/` | 공통 문서화 절차와 템플릿을 정의합니다. |
| `agents/` | 문서 품질 검토 역할을 정의합니다. |
| `hooks/` | 초기에는 사용하지 않습니다. |

상세 구성은 각 디렉터리의 README를 참고합니다.

- `skills/README.md`
- `agents/README.md`
- `hooks/README.md`

## 다른 플러그인과의 경계

| 항목 | 담당 플러그인 |
|---|---|
| 요구사항 명확화/작업 분류 | `common-core` |
| Git/테스트/PR/배포 흐름 | `common-dev-workflow` |
| 보안/운영 안전 | `common-security` |
| 작업 완료 문서/보고/릴리즈 노트 | `common-documentation` |
| 백엔드 구현 상세 | `backend-tools` |
| 프런트엔드 구현 상세 | `frontend-tools` |
| DB 설계/마이그레이션 상세 | `dba-tools` |

## 권장 사용 흐름

```text
작업 내용 수집
→ 문서 유형 선택
→ 문서 초안 작성
→ 검증 상태 명시
→ 문서 품질 리뷰
```

## 운영 원칙

- 확인되지 않은 내용은 `확인 필요` 또는 `미검증`으로 표시합니다.
- 테스트를 실행하지 않았다면 테스트 통과로 작성하지 않습니다.
- 민감정보, 개인정보, 운영 정보는 마스킹하거나 제외합니다.
- 문서 목적에 맞게 필요한 내용만 작성합니다.
- 상세 기술 검토는 직군별 플러그인에 맡기고, 문서화 플러그인은 기록 품질에 집중합니다.
