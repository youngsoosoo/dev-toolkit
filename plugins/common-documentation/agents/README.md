# common-documentation agents

`common-documentation/agents`는 공통 문서의 품질을 검토하는 역할을 정의합니다.

Agent는 문서가 목적에 맞게 작성되었는지, 확인된 내용과 미검증 항목이 구분되어 있는지, 독자가 바로 이해할 수 있는지 검토합니다.

## 구성

| Agent | 파일 | 역할 |
|---|---|---|
| `documentation-reviewer` | `documentation-reviewer.md` | 공통 문서 품질 리뷰어 |

## documentation-reviewer

작업 완료 문서, 주간 보고, 릴리즈 노트, 장애 보고서, 의사결정 기록의 품질을 검토합니다.

검토 기준:

- 문서 목적에 맞는 구조인지
- 완료/미완료/미검증 항목이 구분되어 있는지
- 확인된 사실과 추측이 분리되어 있는지
- 독자가 후속 행동을 이해할 수 있는지
- 민감정보 또는 불필요한 세부 정보가 포함되어 있지 않은지

## 작성 원칙

- 문서의 길이보다 명확성을 우선합니다.
- 검증되지 않은 내용을 확정적으로 표현하지 않습니다.
- 기술 세부사항이 필요한 경우 직군별 플러그인 문서를 참고하도록 안내합니다.
